# 🍳 EasyCook 요구사항 명세서

## SFR-001 레시피 서비스

### SFR-001-01 외부 API 연동 시스템

**상세 요구사항** ○ 식품안전처 조리식품의 레시피 DB 메타정보 Open API와 연동하여 레시피 데이터를 실시간으로 조회할 수 있는 기능을 구현한다. API 호출 시 인증키, 서비스명, 데이터 타입, 요청 범위를 파라미터로 전달하여 XML 또는 JSON 형태의 응답을 받는다.

○ API 응답 데이터의 안정성을 위해 외부 API 호출 실패 시 재시도 로직을 구현하고, 일정 시간 초과 시 사용자에게 적절한 오류 메시지를 표시한다. 또한 API 호출 빈도 제한을 고려하여 적절한 지연 시간을 두고 요청을 처리한다.

○ 외부 API로부터 받은 레시피 데이터를 내부 데이터베이스에 캐싱하여 응답 속도를 개선하고, 동일한 요청에 대해서는 캐시된 데이터를 우선적으로 제공한다. 캐시 데이터는 24시간마다 갱신하여 최신 정보를 유지한다.

### SFR-001-02 레시피 검색 기능

**상세 요구사항** ○ 사용자가 메뉴명을 입력하여 관련 레시피를 검색할 수 있는 기능을 제공하며, 부분 검색과 정확한 검색을 모두 지원한다. 검색 결과는 관련도 순으로 정렬하여 표시하고, 검색어 하이라이팅 기능을 제공한다.

○ 재료명을 기준으로 한 검색 기능을 구현하여 사용자가 보유한 식재료로 만들 수 있는 레시피를 찾을 수 있도록 한다. 여러 재료를 입력할 경우 AND 조건과 OR 조건을 선택할 수 있도록 한다.

○ 요리종류(반찬, 국, 후식 등), 조리방법(볶음, 찜, 구이 등)별 필터링 기능을 제공하여 사용자가 원하는 조건에 맞는 레시피를 쉽게 찾을 수 있도록 한다. 필터는 다중 선택이 가능하며 선택된 조건들은 화면에 표시된다.

### SFR-001-03 레시피 상세 정보 제공

**상세 요구사항** ○ 선택된 레시피의 기본 정보인 메뉴명, 조리방법, 요리종류, 중량, 칼로리를 명확하게 표시하고, 탄수화물, 단백질, 지방, 나트륨 등 주요 영양성분 정보를 구조화된 형태로 제공한다.

○ 레시피의 재료 정보와 만드는 방법을 단계별로 표시하며, 각 단계별 이미지가 있는 경우 함께 제공한다. 만드는 방법은 최대 20단계까지 지원하며, 단계별로 명확하게 구분하여 사용자가 쉽게 따라할 수 있도록 한다.

○ 레시피에 포함된 해시태그 정보를 활용하여 관련 레시피 추천 기능을 제공하고, 저감 조리법 TIP이 있는 경우 별도 영역에 표시하여 건강한 요리 방법을 안내한다.

### SFR-001-04 레시피 목록 관리

**상세 요구사항** ○ 검색된 레시피 목록을 페이지 단위로 표시하며, 페이지당 20개의 레시피를 기본으로 하고 사용자가 10개, 20개, 50개 중 선택할 수 있도록 한다. 목록에는 대표 이미지, 레시피명, 간단한 영양정보, 요리종류를 표시한다.

○ 레시피 목록 정렬 기능으로 최신순, 이름순, 칼로리 높은순/낮은순을 제공하며, 사용자가 선택한 정렬 조건을 세션에 저장하여 다음 검색 시에도 동일한 조건을 유지한다.

○ 무한 스크롤 방식을 지원하여 사용자가 스크롤을 내릴 때마다 추가 레시피를 자동으로 로드하는 기능을 제공한다. 로딩 중에는 적절한 로딩 인디케이터를 표시한다.

### 용어사전

- **레시피**: 식품안전처에서 제공하는 공식 조리법 정보
- **API 캐싱**: 외부 API 응답을 임시 저장하여 성능을 개선하는 기술
- **해시태그**: 레시피의 특성을 나타내는 키워드 태그
- **저감 조리법**: 나트륨이나 당분을 줄인 건강한 조리 방법

## SFR-002 영양정보 서비스

### SFR-002-01 영양정보 API 연동 시스템

**상세 요구사항** ○ 전국통합식품영양성분정보(원재료성식품) 표준데이터 API와 연동하여 식품의 상세 영양성분 정보를 실시간으로 조회할 수 있는 시스템을 구현한다. API 호출 시 필요한 인증 절차를 거치고 안정적인 데이터 수신을 보장한다.

○ 영양정보 API 응답 데이터의 신뢰성을 확보하기 위해 데이터 검증 로직을 구현하고, 비정상적인 데이터나 누락된 필드에 대한 예외 처리를 수행한다. API 호출 실패 시 사용자에게 명확한 안내 메시지를 제공한다.

○ 영양정보 조회 성능 향상을 위해 자주 검색되는 식품 정보를 로컬 데이터베이스에 캐싱하고, 캐시 만료 시간을 설정하여 최신 정보를 유지한다. 동일한 식품에 대한 중복 API 호출을 방지한다.

### SFR-002-02 식품 검색 및 조회 기능

**상세 요구사항** ○ 사용자가 식품명을 입력하여 해당 식품의 영양성분 정보를 검색할 수 있는 기능을 제공한다. 검색은 완전일치와 부분일치를 모두 지원하며, 검색 결과는 관련도 순으로 정렬하여 표시한다.

○ 식품 분류별 브라우징 기능을 구현하여 곡류, 채소류, 과일류, 육류 등 카테고리별로 식품을 탐색할 수 있도록 한다. 각 카테고리 내에서도 세부 분류로 나누어 사용자의 탐색 편의성을 높인다.

○ 영양성분별 필터링 기능을 제공하여 고단백질, 저칼로리, 고섬유질 등 특정 영양 조건에 맞는 식품을 쉽게 찾을 수 있도록 한다. 필터 조건은 다중 선택이 가능하며 조건값 범위를 사용자가 직접 설정할 수 있다.

### SFR-002-03 영양성분 상세 정보 표시

**상세 요구사항** ○ 선택된 식품의 기본 정보(식품명, 분류, 중량 기준)와 함께 칼로리, 탄수화물, 단백질, 지방, 나트륨, 콜레스테롤 등 주요 영양성분을 표 형태로 구조화하여 표시한다.

○ 비타민, 무기질 등 미량 영양소 정보도 상세히 제공하며, 1일 영양섭취기준 대비 해당 식품의 영양소 함량 비율을 백분율로 표시하여 사용자가 영양가를 쉽게 파악할 수 있도록 한다.

○ 영양성분 정보를 시각적으로 표현하기 위해 차트나 그래프를 활용하여 영양소별 함량을 직관적으로 비교할 수 있는 기능을 제공한다. 특히 3대 영양소(탄수화물, 단백질, 지방)의 비율을 원형 차트로 표시한다.

### SFR-002-04 영양정보 비교 기능

**상세 요구사항** ○ 최대 3개의 식품을 선택하여 영양성분을 나란히 비교할 수 있는 기능을 구현한다. 비교 테이블에서는 각 영양소별로 가장 높은 값을 하이라이트 표시하여 차이점을 명확하게 인식할 수 있도록 한다.

○ 비교 대상 식품들의 영양밀도(단위 중량당 영양소 함량)를 계산하여 표시하고, 동일한 칼로리 기준으로 환산한 영양성분 비교 정보도 제공한다.

○ 비교 결과를 이미지나 PDF 형태로 저장하거나 공유할 수 있는 기능을 제공하여 사용자가 영양 상담이나 식단 계획 시 활용할 수 있도록 한다.

### 용어사전

- **영양밀도**: 단위 칼로리당 영양소 함량을 나타내는 지표
- **1일 영양섭취기준**: 건강한 성인이 하루에 섭취해야 할 영양소의 권장량
- **미량 영양소**: 비타민, 무기질 등 소량이지만 필수적인 영양소
- **3대 영양소**: 탄수화물, 단백질, 지방을 통칭하는 용어

## SFR-003 즐겨찾기/저장 서비스

### SFR-003-01 레시피 저장 관리 시스템

**상세 요구사항** ○ 사용자가 관심 있는 레시피를 개인 컬렉션에 저장할 수 있는 기능을 제공하며, 저장 시점에 레시피의 기본 정보와 상세 내용을 사용자 데이터베이스에 기록한다. 저장된 레시피는 원본 데이터 변경과 무관하게 저장 시점의 정보를 유지한다.

○ 저장된 레시피를 사용자 정의 카테고리별로 분류할 수 있는 기능을 구현한다. 기본 카테고리(아침, 점심, 저녁, 간식, 디저트)를 제공하고, 사용자가 직접 카테고리를 생성하고 관리할 수 있도록 한다.

○ 레시피 저장 시 개인적인 메모나 평점을 함께 기록할 수 있는 기능을 제공하여 사용자만의 요리 노트를 작성할 수 있도록 한다. 메모는 500자 이내로 제한하고 평점은 5점 만점으로 평가한다.

### SFR-003-02 영양정보 즐겨찾기 시스템

**상세 요구사항** ○ 자주 찾는 식품의 영양정보를 즐겨찾기로 등록하여 빠르게 접근할 수 있는 기능을 구현한다. 즐겨찾기 등록 시 해당 식품의 핵심 영양정보를 요약하여 저장하고 목록에서 바로 확인할 수 있도록 한다.

○ 즐겨찾기한 식품들을 용도별로 그룹핑할 수 있는 기능을 제공한다. 다이어트용, 근력운동용, 건강관리용 등 사용자가 직접 그룹을 생성하고 식품을 분류할 수 있도록 한다.

○ 즐겨찾기 목록에서 식품 간 영양성분 간단 비교 기능을 제공하여 사용자가 빠르게 영양가를 비교하고 선택할 수 있도록 한다. 비교는 칼로리, 단백질, 탄수화물, 지방 등 주요 4가지 영양소를 기준으로 한다.

### SFR-003-03 개인 컬렉션 관리 기능

**상세 요구사항** ○ 저장된 레시피와 즐겨찾기 영양정보를 통합하여 관리할 수 있는 개인 대시보드를 제공한다. 최근 저장한 항목들을 시간순으로 표시하고, 카테고리별 저장 개수 통계를 제공한다.

○ 저장된 데이터에 대한 검색 기능을 구현하여 사용자가 본인이 저장한 레시피나 영양정보를 빠르게 찾을 수 있도록 한다. 검색은 제목, 재료, 메모 내용을 대상으로 하며 통합 검색과 카테고리별 검색을 지원한다.

○ 컬렉션 내용을 백업하고 복원할 수 있는 기능을 제공하여 사용자 데이터의 안전성을 보장한다. 백업 데이터는 JSON 형태로 내보내기가 가능하며, 다른 기기에서 로그인 시 데이터를 동기화할 수 있다.

### SFR-003-04 컬렉션 공유 및 활용 기능

**상세 요구사항** ○ 사용자가 만든 레시피 컬렉션을 다른 사용자와 공유할 수 있는 기능을 구현한다. 공유할 컬렉션을 선택하면 고유한 공유 링크를 생성하고, 링크를 받은 사람은 해당 컬렉션을 열람하거나 자신의 계정으로 복사할 수 있다.

○ 인기 있는 공개 컬렉션을 추천하는 기능을 제공하여 다른 사용자들이 만든 유용한 레시피 모음을 발견할 수 있도록 한다. 추천은 저장 횟수, 평점, 최신성을 종합하여 순위를 결정한다.

○ 저장된 레시피 기반으로 주간 식단표를 자동 생성하는 기능을 제공한다. 사용자가 선호하는 카테고리와 영양 목표를 설정하면 저장된 레시피 중에서 적절한 조합을 제안하여 균형잡힌 식단을 구성할 수 있도록 한다.

### 용어사전

- **컬렉션**: 사용자가 저장한 레시피와 영양정보를 모은 개인 자료집
- **카테고리**: 저장된 데이터를 분류하기 위한 그룹 단위
- **동기화**: 여러 기기 간 사용자 데이터를 일치시키는 과정
- **공유 링크**: 컬렉션을 다른 사람과 공유하기 위한 고유 URL
